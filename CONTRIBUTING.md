# Contributing to ERETA

First off, thank you for considering contributing to ERETA! 

## Getting Started

1. **Fork** the repository on GitHub.
2. **Clone** the project to your own machine.
3. **Commit** changes to your own branch.
4. **Push** your work back up to your fork.
5. Submit a **Pull Request** so that we can review your changes.

## Guidelines

- **No large datasets:** Please do not commit large dataset files (like `.tiff`) or binary files unnecessarily. Ensure `Raw_Footage_SIH25.mov` and data directories remain ignored.
- **No secrets:** Never commit `.env` or `config.json` files containing real API keys or database URLs.
- **Run formatters:** Ensure your code is properly formatted before submitting a PR (e.g., standard Python formatters for the ML/backend, and Prettier for the Next.js frontend).
- **Descriptive Commits:** Write clear and descriptive commit messages.

## Branching Naming Convention

- Features: `feature/your-feature-name`
- Bugfixes: `bugfix/issue-description`
- Docs: `docs/what-you-changed`

Thank you for helping us improve ERETA!
